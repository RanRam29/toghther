# WP7 — הכנה ל-Launch

> **בעלים:** שניהם · **תלוי ב:** WP1–WP6 · **אבן דרך:** 7
> **מטרה:** מוכנות ל-beta סגורה בעיר ההשקה (D18 — גוש דן מזרחי, לאישור בעל מוצר).
> קרא: `docs/TESTING-STRATEGY.md` (צ'קליסט E2E §4) · `master_spec` §12 · `product/01-DECISIONS.md` D18.

## משימות
1. **Seed מציאותי** — עדכון `supabase/seed.sql`: 50+ פרופילי משלבות מאומתות באזור ההשקה (מפוזרות גיאוגרפית), הורים+ילדים לדוגמה בכל אבחנה, matches פעילים עם היסטוריית יומנים — כדי שההתאמות לא יהיו ריקות ביום הראשון (פתרון ה-cold-start מהמסמך המנחה).
2. **CI (GitHub Actions)** — אם לא הוקם ב-WP0: supabase start + מיגרציות מאפס + `supabase test db` + tsc + lint על כל PR (docs/TESTING-STRATEGY §5).
3. **E2E ידני מלא** — צ'קליסט TESTING-STRATEGY §4 עד הסוף, בעברית RTL, iOS+Android, כולל revoke ומצב offline.
4. **סקירת אבטחה סופית** — כל C1–C4 סגורים ומכוסים בבדיקות; מפתחות ב-secrets בלבד; buckets פרטיים; ולידציית Security-DoD (docs/SECURITY-GUIDELINES §10).
5. **הקשחת אדמין** — MFA לחשבונות אדמין (AUTH-SPEC §4); CORS ל-origins מוגדרים במקום `*`.
6. **EAS Build + TestFlight/Internal** — הגדרת `eas.json`, secrets ל-build, בילדים ל-iOS+Android, הפצה פנימית.
7. **ייעוץ משפטי (חוסם launch, לא קוד)** — אימות תעודות יושר, מדיניות פרטיות ותנאי שימוש, שמירת audit. פנייה לבעל המוצר.
8. **גיוס supply בפועל** — לא קוד: 50 המשלבות הראשונות בעיר ההשקה (סיכון cold-start מהמסמך המנחה). להתחיל מוקדם, במקביל.

## Definition of Done (Go/No-Go ל-beta)
- [ ] הורה חדש רואה 3+ התאמות אמיתיות ביום הראשון (seed)
- [ ] כל צ'קליסט ה-E2E ירוק; CI ירוק על main
- [ ] 0 ממצאי אבטחה פתוחים; כל תרחישי C1–C4 בבדיקות
- [ ] MFA אדמין פעיל; CORS מוגבל
- [ ] בילדים זמינים לבודקים; onboarding עובד על מכשיר אמיתי
- [ ] אישור משפטי + עיר השקה מאושרת (בעל המוצר)
