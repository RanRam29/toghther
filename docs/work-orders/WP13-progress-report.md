## עיקרון-על (D53, מחייב)

הדוח הוא **זכות עיון של ההורה** — לא ייצוא ולא כלי אדמין. הוא **דוח התקדמות, לא תיק רפואי**, ולכן:

**אסור שיופיע בדוח, בשום שדה, בשום גרסה:**
- אבחנה — `category`, `secondary_category`, `functioning_level`, `communication_verbal`, `needs`
- אף שדה מ-`child_details`
- `daily_logs.notes` (טקסט חופשי יומי) ו-`ai_summary`/`ai_strategy` (שדות מורשת של D30)
- טלפון/פרטי קשר של המשלבת, קואורדינטות, מזהי UUID גולמיים

**מותר:** שם פרטי וגיל של הילד, שם תצוגה של המשלבת, ותקופות — ולצידם מספרים מצטברים בלבד: נוכחות, ספירת דיווחים, ממוצעי מצב רוח ומדדים.

"רגעי היום" (D51) **לא** נכללים בגרסה 1 — השדה שייך ל-WP12 שטרם נבנה. הפלט כולל `report_version: 1`; אחרי WP12 תצא גרסה 2 שמוסיפה אותם.

## 1. צד שרת — מיגרציה אחת (Antigravity)

### 1.1 ‏RPC ‏`get_child_progress_report(p_child_id uuid, p_from date, p_to date) RETURNS jsonb`

**תבנית חובה:** `SECURITY DEFINER` + ‏`SET search_path = public, pg_temp` **באותה הצהרה** (הכלל מ-2026-07-13 — לא בירושה). `GRANT EXECUTE` ל-`authenticated` בלבד, `REVOKE` מ-`public`.

**הרשאה (לפני כל דבר אחר):** הקורא הוא ההורה של הילד — `children.parent_id = auth.uid()` **או** `children.secondary_parent_id = auth.uid()` (מודל D31 הקיים מ-`20260712210000`). הילד לא מחוק (`deleted_at IS NULL`). כל כישלון ⇒ חריגה אחידה, בלי להבחין בין "לא קיים" ל"לא שלך" (מניעת בירור קיום).

**ולידציה:** `p_from <= p_to`; טווח מקסימלי 366 יום; אחרת חריגה.

**חישוב** — על כל ה-matches של הילד שחופפים לטווח (כולל שהסתיימו — היסטוריה היא הערך):

```jsonc
{
  "report_version": 1,
  "child": { "first_name": "...", "age": 6 },
  "period": { "from": "2026-01-01", "to": "2026-07-01" },
  "matches": [
    {
      "professional_name": "דנה",            // שם תצוגה בלבד
      "started_at": "2026-02-01",
      "ended_at": null,                       // null = פעיל
      "metrics_keys": ["regulation", "transitions", "social_initiative"],
      "attendance": { "days_attended": 74, "days_off": 6 },
      "reporting":  { "logs_count": 70, "mood_avg": 3.8 },
      "weeks": [                              // דלי שבועי לגרפים
        { "week_start": "2026-02-01", "logs": 4, "mood_avg": 3.5,
          "metrics_avg": { "regulation": 3.2, "transitions": 2.8, "social_initiative": 3.9 } }
      ]
    }
  ],
  "totals": { "days_attended": 74, "logs_count": 70, "mood_avg": 3.8 }
}
```

**דיוק חובה:**
- `days_attended` = תאריכים ייחודיים של check-ins **תקפים בלבד** (`is_valid = true`), לפי `(created_at AT TIME ZONE 'Asia/Jerusalem')::date` — לא UTC (check-in של 23:30 שייך ליום הנכון בישראל).
- `days_off` = ספירת שורות `match_days_off` בטווח (D46) — מוצג בנפרד, לא נספר כהיעדרות.
- ממוצעי מדדים מ-`daily_logs.metrics` (JSONB) — רק על שלושת המפתחות של ה-match; מפתח חסר ביום נתון לא נספר כאפס.
- שבוע מתחיל בראשון (D21).

**מה ה-RPC לא עושה:** לא כותב ל-`audit_log` (פעולה של בעל המידע, לא של אדמין); לא מוגבל קצב (קריאה בלבד על נתוני עצמו). אירועי האנליטיקה נרשמים בצד האפליקציה (סעיף 3).

### 1.2 אחרי המיגרציה
`npm run types:generate` + הודעה בלוח באותו סבב (שינוי חתימות → Cursor תלוי בזה).

## 2. בדיקות pgTAP — ‏`supabase/tests/wp13_progress_report_test.sql` (Antigravity)

1. **הרשאה:** הורה של ילד אחר ⇒ חריגה · משלבת (גם עם match פעיל על הילד!) ⇒ חריגה · אנונימי ⇒ חריגה.
2. **הורה משני** (seed עם `secondary_parent_id`) ⇒ מקבל דוח תקין.
3. **ולידציה:** טווח הפוך ⇒ חריגה · טווח > 366 יום ⇒ חריגה · ילד עם `deleted_at` ⇒ חריגה.
4. **נכונות:** `logs_count` ו-`days_attended` תואמים ספירה ישירה על ה-seed; check-in עם `is_valid = false` לא נספר; יום עם שורת `match_days_off` מופיע ב-`days_off` ולא ב-`days_attended`.
5. **אי-דליפה (החשובה ביותר):** הפלט כטקסט אינו מכיל את ערכי ה-seed של `category`/`needs`/`notes`/`phone` ואף מחרוזת מ-`child_details`. (אותה שיטה כמו `wp10_reports_test.sql`.)

## 3. אפליקציה — מסך "דוח התקדמות" (Cursor, אחרי סעיפים 1–2)

- **כניסה:** קישור "דוח התקדמות" במסך העבודה המשותפת (S-PAR-07, ליד "מי צפה בתיק") + מפרופיל הילד (S-PAR-09).
- **בורר תקופה:** ארבע קביעות — 30 יום · 3 חודשים · מההתחלה · טווח מותאם. ברירת מחדל: 3 חודשים.
- **תצוגה:** כרטיס סיכום (ימי ליווי, דיווחים, מצב רוח ממוצע) → גרף מגמת מצב רוח שבועי (TrendChart הקיים) → ממוצעי שלושת המדדים לפי שבוע → פירוט תקופות ליווי. עברית קודם, RTL, תאריכים he-IL (D21).
- **הפקה ושיתוף:** `expo-print` ל-PDF → share sheet. **לפני השיתוף — sheet אזהרה חובה:** "הדוח כולל מידע אישי על {שם}. שתפי אותו רק עם מי שאת סומכת עליו." (נוסח סופי לפי 06-COPY-TONE; כפתור המשך אחד, לא אשף).
- **מצב ריק:** פחות מ-3 ימי דיווח בתקופה ⇒ תבנית 3 החלקים ("עוד אין מספיק נתונים כדי להרכיב דוח — נחזור לכאן אחרי כמה ימי ליווי"), בלי שגיאה.
- **אנליטיקה (להוסיף ל-08-ANALYTICS):** `progress_report_generated` ‏(period_days) · `progress_report_shared`. בלי תוכן הדוח באירוע.
- **אין שמירת קובץ אוטומטית** — ה-PDF נוצר לרגע השיתוף בלבד; לא נכתב ל-Storage של המערכת.
