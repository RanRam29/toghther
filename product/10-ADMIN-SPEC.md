> **דרישת בעל המוצר (2026-07-07): יכולת לנהל את כל המערכת מאזור מנהל.** מעדכן את ההחלטה הקודמת ("Studio בלבד ב-MVP") — ראו D23–D26 ב-[01-DECISIONS.md](01-DECISIONS.md).

## הכרעה ארכיטקטונית

**אזור Staff (מנהל + מפקח) הוא route group ‏`(staff)` באפליקציית ה-Expo, בשימוש דרך Web** (`npx expo start --web` / deploy web). נימוקים:
1. המסמך המנחה כבר קבע: "קוד אחד → iOS + Android + **Web Admin**" — זו הכוונה המקורית.
2. אותו design system, אותם types, אותם hooks — אפס תשתית חדשה.
3. ההרשאה נאכפת בשלוש שכבות: route guard (‏role) → RLS policies → RPC-י staff ייעודיים.

**עקרונות ברזל:**
- **כל פעולת staff דרך RPC ייעודי** — לא UPDATE ישיר. כל RPC כותב ל-`audit_log` ‏(prefix ‏`admin_*` או `supervisor_*`).
- **אדמין לא רואה TIER 3 כברירת מחדל.** מסמכים רפואיים ותוכן יומנים — רק במסלול "צפייה מנומקת".
- **אין מחיקות מה-UI.** השבתה/השהיה בלבד.
- חשבונות staff (אדמין, מפקח) נוצרים ידנית ב-DB.

## תפקיד מפקח (D26) — אימות משלבות בלבד

| | מפקח (`supervisor`) | אדמין (`admin`) |
|--|---------------------|-----------------|
| אימות | ✅ תור + שיוך + אישור/דחייה | ✅ + שחרור שיוך |

| ילדים / matches / config | ⛔ | ✅ (לפי מסך) |

### S-STF-01 · תור אימות למפקח (= S-ADM-02 עם הגבלות)
- **תור פנוי:** `submitted` ללא `assigned_supervisor_id` — כפתור "לקחת לטיפול".
- **התור שלי:** משויכות אלי; רק אלה ניתנות לביקורת/הכרעה.
- מסך ביקורת: כמו S-ADM-02; **טלפון מוסתר** עד `supervisor_log_document_view` ×3 + `supervisor_reject_document` עם סיבה.
- אדמין רואה את אותם מסכים + טלפון תמיד + שחרור שיוך.

## המסכים (אדמין)

### S-ADM-01 · דשבורד (בית)
מדדי הפלטפורמה בזמן אמת: משלבות מאומתות / בהמתנה (+SLA חורג באדום) · הורים וילדים פעילים · בקשות פתוחות ו-conversion המשפך (מ-analytics_events) · matches פעילים · check-ins היום · שאלונים שמולאו היום. כל מספר לחיץ → מסך הרשימה המסונן.

### S-ADM-02 · תור אימות ⭐ (הראשון שנבנה)
- רשימת `verified='submitted'` לפי ותק בתור, עם חיווי SLA (ירוק <1 יום, צהוב 1–2, אדום >2).
- מסך משלבת: פרטי פרופיל + שלושת המסמכים ב-viewer מוטמע (signed URLs קצרי מועד) + צ'קליסט האימות (מה-runbook, אינטראקטיבי — כל סעיף חובה לסימון).
- פעולות: "אשר ✓" (נעול עד השלמת הצ'קליסט) / "דחה מסמך" (בחירת מסמך + סיבה חובה) → RPC → push אוטומטי למשלבת.
- בדיקת ביו: שדה הביו מוצג עם הדגשת דפוסי טלפון/אימייל (regex) — אכיפת "אין פרטי קשר בפרופיל".

### S-ADM-03 · משתמשים
טבלת הורים ומשלבות: חיפוש, סינון לפי תפקיד/סטטוס/אזור. פרופיל משתמש: פרטים, פעילות אחרונה, matches, דגלים. פעולות: השהיית חשבון (+סיבה) / שחזור / איפוס OTP / הוספת הערת אדמין. השהיה = חסימת כניסה + הסתרה מכל התאמות ו-browse.

### S-ADM-04 · ילדים ובקשות (תצוגת TIER 1 בלבד)
רשימת כרטיסי ילדים (ללא child_details!) וסטטוס בקשות. שימוש: איתור תקיעות במשפך (בקשות שפג תוקפן בהמוניהן = בעיית supply באזור), הסרת כרטיס בעייתי (unpublish + סיבה).

### S-ADM-05 · ‏Matches ואופרציה
matches פעילים עם בריאות תפעולית: ימים מאז check-in אחרון, רצף שאלונים. סינון "מודאגים" (ללא פעילות 3+ ימים) — הבסיס האנושי לחמ"ל לפני שהוא אוטומטי ב-v1.5. פעולות: הערת אדמין, פנייה יזומה (מסומנת לטיפול טלפוני — אין הודעות בפלטפורמה, D9).

### S-ADM-06 · ‏Audit Log
צפייה בכל ה-audit: סינון לפי משתמש/משאב/פעולה/תאריך. כולל פעולות אדמין עצמן (`admin_*`). קריאה בלבד.

### S-ADM-07 · תפעול ותוכן
- **קטלוג מדדים** (07-METRICS-CATALOG): עריכת תוויות, הוספת מדד, שיוך לקטגוריות — בלי release.
- **פרמטרי מערכת** (`system_config` key-value): רדיוס geofence, ימי תפוגת בקשה, מכסת בקשות, עיר השקה — הערכים מ-D7/D8 חיים כאן, לא בקוד.
- **צפייה ב-analytics**: המשפכים מ-08-ANALYTICS-EVENTS כטבלאות פשוטות.

## דרישות backend נגזרות (Antigravity)

1. RPC-ים: `admin_verify_professional(pro_id, checklist jsonb)` · `admin_reject_document(doc_id, reason)` · `admin_suspend_user(user_id, reason)` · `admin_restore_user` · `admin_unpublish_child(child_id, reason)` · `admin_log_reasoned_view(resource, id, reason)` — כולם בודקים `get_user_role()='admin'` וכותבים audit.
2. טבלאות: `system_config` ‏(key, value jsonb, updated_by) · `admin_notes` ‏(target_user_id, note, created_by) · עמודת `suspended_at` ב-profiles.
3. שדה `checklist` נשמר עם האימות — ראיה משפטית שהבדיקה בוצעה.
4. בדיקות pgTAP: משתמש שאינו אדמין נחסם מכל RPC אדמיני; אדמין לא קורא child_details בלי `admin_log_reasoned_view`.
